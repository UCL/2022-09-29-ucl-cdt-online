Git

Git is a version control system that lets you track who made changes to what when and has options for easily updating a shared or public version of your code on github.com. You will need a supported web browser.

You will need an account at github.com for parts of the Git lesson. Basic GitHub accounts are free. We encourage you to create a GitHub account if you don't have one already. Please consider what personal information you'd like to reveal. For example, you may want to review these instructions for keeping your email address private provided at GitHub.

  • Windows
  • MacOS
  • Linux

Git should be installed on your computer as part of your Bash install (see the Shell installation instructions).

Proceed to install Github CLI by downloading the installer.

For macOS, install Git for Mac through Xcode.

  1. Open a terminal window

  2. Type 

    git --version
    • If you do not have git, it will automatically asks you whether you want to install Xcode. Confirm the installation and you will get git installed.
    • If you have git, then it should display the version of git you've got installed.

The version provided by Apple may be a few versions behind, but that's not a problem for this course if it's newer than version 2.28. If you need or want to obtain a newer version, then install git with a different method.

Once you've installed git, proceed to install Github CLI. The easiest way is through homebrew. First you need to install homebrew:

  1. Open a terminal window

  2. Type 

    /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

    Follow the installation prompts

  3. Once homebrew is installed, you can then install Github's CLI

    brew install gh

If Git is not already available on your machine you can try to install it via your distro's package manager. For Debian/Ubuntu run sudo apt-get install git and for Fedora run sudo dnf install git.

Then proceed to install Github's CLI following the option according to your distribution.
